# Glossary

| Term | Definition |
|---|---|
| **BMS** | Basic Mapping Support — CICS facility for defining 3270 screen maps |
| **BZUCFG** | zUnit runner configuration file (XML) that pairs a test case program with its playback file |
| **CBSA** | CICS Bank Sample Application — this project |
| **CICS** | Customer Information Control System — IBM transaction-processing middleware |
| **COMMAREA** | Communication area — fixed-length data block passed between CICS programs via EXEC CICS LINK |
| **Copybook** | COBOL include file (`.cpy`) defining shared data layouts |
| **DBB** | IBM Dependency Based Build — build framework for z/OS source |
| **DBCG** | DB2 subsystem name used by CBSA |
| **DSECT** | Dummy section — COBOL/Assembler structure generated from a BMS map for use in programs |
| **ENQ/DEQ** | EXEC CICS ENQ/DEQ — CICS resource enqueue/dequeue for serialization |
| **HLQ** | High-Level Qualifier — the prefix for z/OS dataset names (e.g., `GITLAB.CBSA`) |
| **Named Counter** | CICS shared counter used for sequential number generation (account/customer numbers) |
| **PCBSA** | DB2 package collection ID used for CBSA bind |
| **PROCTRAN** | Processing Transaction — CBSA's audit log table (`IBMUSER.PROCTRAN`) |
| **SAR / AAR** | Service/API Archive — deployable artifacts for z/OS Connect EE |
| **Sort Code** | 6-digit bank identifier (e.g., `987654`) defined in `SORTCODE.cpy` |
| **SSI** | System Status Information — 8-char field in load module used to store the abbreviated Git hash |
| **UoW** | Unit of Work — a DB2 transaction (commit scope) |
| **zAppBuild** | IBM open-source Groovy build framework built on DBB |
| **zUnit** | IBM Z Automated Unit Testing Framework — COBOL/PL/I unit test runner |
| **z/OS Connect EE** | z/OS Connect Enterprise Edition — REST API gateway for CICS and IMS |
